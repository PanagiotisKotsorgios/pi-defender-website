Installation Guide

PI-DEFENDER requires a Linux-based system with kernel version 4.15 or later. The following dependencies must be installed:

# Required dependencies
$ sudo apt install build-essential libpcap-dev libssl-dev python3-dev

# Debian/Ubuntu installation
$ sudo apt install ./pi-defender_1.0.2_amd64.deb

# RHEL/CentOS installation
$ sudo yum install pi-defender-1.0.2-1.x86_64.rpm

# Verify installation
$ pi-defender --version
PI-DEFENDER v1.0.2 (build 2025-03-15)

Post-Installation Steps

After installation, you need to configure the service and enable it to start on boot:

# Start the service
$ sudo systemctl start pi-defender

# Enable on boot
$ sudo systemctl enable pi-defender

# Check service status
$ sudo systemctl status pi-defender

Configuration Reference

PI-DEFENDER uses a YAML configuration file located at /etc/pi-defender/config.yaml. Below are the main configuration options:

# Example config.yaml
network_interface: eth0
scan_interval: 300
alert_threshold: medium
log_level: info
email_alerts: true
email_address: admin@example.com
whitelist:
- 192.168.1.100
- 192.168.1.101

Configuration Options

Option Type Default Description
network_interface string eth0 Network interface to monitor
scan_interval integer 300 Scan interval in seconds
alert_threshold string medium Alert sensitivity (low, medium, high)
log_level string info Logging verbosity (debug, info, warning, error)
email_alerts boolean false Enable email notifications

CLI Usage

The PI-DEFENDER command-line interface provides full control over the security system. Here are the primary commands:

# Show help information
$ pi-defender --help

# Start in foreground with debug logging
$ pi-defender start --foreground --log-level debug

# Check current status
$ pi-defender status

# View real-time monitoring
$ pi-defender monitor

# Generate a security report
$ pi-defender report --output report.pdf

Common Command Examples

# Scan network with custom parameters
$ pi-defender scan --interface wlan0 --intensity high

# Add IP to whitelist
$ pi-defender whitelist add 192.168.1.50

# View blocked IP addresses
$ pi-defender blocklist show

# Update threat definitions
$ pi-defender update

API Documentation

PI-DEFENDER provides a REST API for integration with other systems. The API runs on port 8080 by default.

Authentication

All API requests require an API key in the header:

Authorization: Bearer your_api_key_here

Endpoints

# Get system status
GET /api/v1/status

# Start a scan
POST /api/v1/scan
{ "intensity": "high", "targets": ["192.168.1.0/24"] }

# Get scan results
GET /api/v1/results?scan_id=12345

# Manage whitelist
POST /api/v1/whitelist
{ "action": "add", "ip": "192.168.1.100" }

Example Usage with curl

# Get current status
$ curl -H "Authorization: Bearer API_KEY" \
http://localhost:8080/api/v1/status

# Start a new scan
$ curl -X POST -H "Authorization: Bearer API_KEY" \
-H "Content-Type: application/json" \
-d '{"intensity": "medium"}' \
http://localhost:8080/api/v1/scan

Compliance Guide

PI-DEFENDER helps organizations meet various security compliance requirements. Below are the supported frameworks:

GDPR

Helps meet requirements for personal data protection and breach notification timelines.

ISO 27001

Supports Annex A controls for network security management and access control.

PCI DSS

Meets requirements for network monitoring and intrusion detection (Req 11.4).

HIPAA

Supports technical safeguards for access control and audit controls (164.312).

Configuration for Compliance

To enable compliance reporting, add these settings to your config.yaml:

compliance:
frameworks:
- gdpr
- pci_dss
report_interval: monthly
retention_period: 7 years

Generating Compliance Reports

# Generate GDPR compliance report
$ pi-defender compliance report --framework gdpr --output gdpr-report.pdf

# Generate all compliance reports
$ pi-defender compliance report --all

Troubleshooting

Common issues and solutions for PI-DEFENDER:

1

Service fails to start

Solution: Check system logs with journalctl -u pi-defender. Ensure all dependencies are installed.

2

No network traffic detected

Solution: Verify correct network interface in config.yaml. Run pi-defender interfaces to list available interfaces.

3

False positive alerts

Solution: Adjust alert_threshold to 'low'. Add legitimate IPs to whitelist.

4

High CPU usage

Solution: Increase scan_interval in configuration. Check for network loops.

5

Email alerts not sending

Solution: Verify SMTP settings. Test with pi-defender test-email command.

Diagnostic Commands

# Run system diagnostics
$ pi-defender diagnose

# Generate debug report
$ pi-defender debug-report

# Check dependency versions
$ pi-defender versions
